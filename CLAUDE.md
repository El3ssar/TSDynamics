This file documents the project's architecture, conventions, and patterns so we can work together efficiently on any change.

## Project overview

**TSDynamics** is a Python library for studying dynamical systems. It provides:

- Compiled ODE integration via JiTCODE (`DynSys`)
- Compiled DDE integration via JiTCDDE (`DynSysDelay`)
- Discrete map iteration (`DynMap`)
- Lyapunov spectrum computation for all three
- Visualization (static plots + animations)
- Time-step estimation utilities (curvature, frequency, sagitta)

The philosophy: **you write the math; the library handles compilation, tolerances, and output grids**. The API surface is intentionally small.

**Author:** Daniel Estevez  
**Python:** ≥ 3.12  
**Package manager:** uv (recommended)

## Base class hierarchy

```
BaseDyn  (base/base.py)
├── DynSys        (base/ode_base.py)   — ODE via JiTCODE
├── DynSysDelay   (base/dde_base.py)   — DDE via JiTCDDE
└── DynMap        (base/map_base.py)   — discrete map
```

### `BaseDyn` (base/base.py)

Shared parameter and IC management for all system types.

**Constructor:** `__init__(n_dim=None, params=None, initial_conds=None)`

- `params` dict is the source of truth; each key is also exposed as an attribute
- `__setattr__` syncs attribute writes back into `params`
- `initial_conds` is stored as-is; if `None` at integrate time, random `U[0,1)^n_dim` is used

**Key method:**
```python
generate_timesteps(dt=0.02, steps=None, final_time=1.0) -> np.ndarray
```
- If `steps` given → `steps` takes precedence over `final_time`
- Always includes the endpoint
- Returns a 1D float64 array starting at 0.0

### `DynSys` (base/ode_base.py)

ODE integration via JiTCODE (compiles to C).

**Subclass contract:**
```python
class MyODE(DynSys):
    params = {"a": 1.0, "b": 2.0}   # class-level defaults
    n_dim = 2

    @staticmethod
    def _rhs(y, t, a, b):           # params passed as kwargs (sorted by name)
        return (a * y(0) - y(1), b * y(0) + y(1))

    @staticmethod                    # optional
    def _jac(y, t, a, b):
        return [[a, -1], [b, 1]]
```

- `y(i)` — JiTCODE symbol for state component i
- `t` — JiTCODE symbol for time
- Return a **sequence of length `n_dim`** using symbolic expressions only (no NumPy)
- `_jac` is optional; JiTCODE skips it if it returns `NotImplemented`

**Integration:**
```python
integrate(
    dt=0.02,
    steps=None,
    final_time=100.0,
    initial_conds=None,
    method="RK45",          # see integrator map below
    rtol=1e-6,
    atol=1e-9,
    **kwargs                # forwarded to set_integrator
) -> (t_eval, y_eval)       # shapes: (T,), (T, n_dim)
```

Integrator name map (case-insensitive aliases):

| Alias | JiTCODE integrator |
|---|---|
| `None`, `"auto"`, `"RK45"`, `"dopri5"` | `dopri5` |
| `"dop853"`, `"DOP853"` | `dop853` |
| `"lsoda"`, `"LSODA"` | `lsoda` |
| `"vode"`, `"VODE"` | `vode` |

Internal stepping is always adaptive; output is evaluated on the CubicHermiteSpline for any `t_eval[k] <= ode.t`.

**Lyapunov spectrum:**
```python
lyapunov_spectrum(
    dt=0.1,
    final_time=200.0,
    initial_conds=None,
    n_lyap=None,            # defaults to n_dim
    burn_in=50.0,
    method="dopri5",
    rtol=1e-6,
    atol=1e-9,
    **integrator_kwargs
) -> np.ndarray             # shape (n_lyap,), largest first
```

Uses `jitcode_lyap`; averages local LEs with time-weights equal to elapsed time between samples.

### `DynSysDelay` (base/dde_base.py)

DDE integration via JiTCDDE (compiles to C).

**Subclass contract:**
```python
class MyDDE(DynSysDelay):
    params = {"tau": 1.5, "k": 2.0}
    n_dim = 1

    @staticmethod
    def _rhs(y, t, k, tau):
        return [k * y(0, t - tau) - y(0)]
```

- `y(i, t - tau)` — delayed access
- Return a **list** of length `n_dim`

**Important:** Global `UserWarning` about "target time is smaller than the current time" is suppressed at module import (expected and harmless during spline evaluation).

**Integration:**
```python
integrate(
    dt=0.02,
    steps=None,
    final_time=100.0,
    initial_conds=None,
    rtol=1e-3,              # DDEs often need looser tolerances
    atol=1e-3,
    history=None,           # callable h(s) -> sequence for s <= 0
    **kwargs                # forwarded to set_integration_parameters
) -> (t_eval, y_eval)
```

- `history=None` → constant past from `initial_conds`
- `history` callable receives `s <= 0`, must return sequence of length `n_dim`
- Sets `dde.initial_discontinuities_handled = True` (suppresses JiTCDDE discontinuity warnings)
- `t=0` value is taken from `history(0.0)` or `initial_conds`

**Lyapunov spectrum:**
```python
lyapunov_spectrum(
    dt=0.1,
    final_time=200.0,
    initial_conds=None,
    n_lyap=1,               # user chooses; DDEs have infinitely many
    history=None,
    burn_in=50.0,
    rtol=1e-6,
    atol=1e-9,
    **integration_kwargs
) -> np.ndarray             # shape (n_lyap,)
```

Uses `jitcdde_lyap`; **must use the returned weight** (3rd element of tuple) for averaging — not elapsed time. Only non-zero-weight steps are included.

### `DynMap` (base/map_base.py)

Discrete map iteration with QR-based Lyapunov spectrum.

**Subclass contract:**
```python
class MyMap(DynMap):
    params = {"a": 1.4, "b": 0.3}
    n_dim = 2

    @staticjit           # numba-accelerated (see utils/general.py)
    def _rhs(X, a, b):
        x, y = X
        return (1 - a*x**2 + y, b*x)

    @staticjit
    def _jac(X, a, b):
        x, y = X
        return [[-2*a*x, 1], [b, 0]]
```

- `_rhs(X, *params)` and `_jac(X, *params)` receive params as **positional args in dict-insertion order**
- Return array-like of length `n_dim`

**Iteration:**
```python
iterate(
    initial_conds=None,
    steps=1000,
    max_retries=10          # auto-retries with new random IC on divergence
) -> (time_indices, trajectory)  # shapes: (steps,), (steps, n_dim)
```

Note: `time_indices = np.arange(steps)` (integer array, not float).

**Lyapunov spectrum:**
```python
lyapunov_spectrum(
    y0=None,
    steps=1000,
    num_exponents=None,     # defaults to n_dim
    perturbation_scale=1e-8,
    reorthonormalize_interval=1
) -> np.ndarray
```

Uses QR decomposition of the tangent-space propagator (Jacobian product) along the trajectory from `iterate`.

## Utilities

### `@staticjit` (utils/general.py)

Decorator that applies `numba.njit` to a static method. Degrades gracefully if numba is unavailable.

```python
from tsdynamics.utils.general import staticjit

@staticjit
def _rhs(X, a, b): ...
```

Use this for `_rhs` and `_jac` in `DynMap` subclasses.

### Time-step estimation utilities

Three independent approaches; all take a raw trajectory and return a dataclass.

#### `estimate_curvature_timestep` (utils/curvature_dt.py)

Geometry-based: perpendicular-to-velocity acceleration component.

```python
from tsdynamics.utils.curvature_dt import estimate_curvature_timestep

result = estimate_curvature_timestep(
    y,            # (T, d) trajectory
    dt0,          # original sample spacing
    epsilon,      # acceptable interpolation error
    percentile=95.0,
    nu=1.0,
    return_details=True
)
# result.delta_t  — recommended Δt
# result.stride   — integer subsampling stride
# result.kappa_percentile
# result.predicted_error
# result.indices  — subsample indices into original array
```

#### `estimate_dt_from_spectrum` (utils/frequency_dt.py)

Spectral: Welch PSD → highest significant frequency.

```python
from tsdynamics.utils.frequency_dt import estimate_dt_from_spectrum

result = estimate_dt_from_spectrum(
    y,            # (T, d) trajectory
    dt0,          # original sample spacing
    q_power=0.99,
    snr_db=8.0,
    smooth_bins=8,
    safety=0.9,
    guard_frac=0.9,
    min_bins=64,
    enforce_coarsening=True
)
# result.delta_t   — Δt* = safety / (2 * f_h)
# result.f_hz      — highest significant frequency
# result.converged — False if degenerate spectrum
# result.notes     — diagnostic string
```

#### `estimate_dt_from_sagitta` (utils/sagitta_dt.py)

Geometric sagitta: perpendicular distance from point to chord between neighbors.

```python
from tsdynamics.utils.sagitta_dt import estimate_dt_from_sagitta

result = estimate_dt_from_sagitta(
    y,            # (T,) or (T, d) trajectory
    dt0,          # original sample spacing
    epsilon=0.05, # max acceptable sagitta (relative for 1D embedded)
    percentile=95.0,
    coarsen_only=True,
    min_points_per_segment=3,
    search_growth=1.5
)
# result.delta_t  — stride * dt0
# result.stride   — integer subsampling stride m*
# result.indices  — subsample indices
```

For 1D input, auto-applies Takens delay embedding using AMI lag and FNN dimension estimates.

## Code conventions

### Style

- **Formatter:** `black`, line length 100
- **Linter:** `ruff` (zero warnings target)
- **Type checking:** `mypy` where feasible
- **Docstrings:** NumPy convention (`pydocstyle` configured in `pyproject.toml`)
- **Commit messages:** Conventional Commits (`feat:`, `fix:`, `docs:`, `refactor:`, `test:`)

Run before committing:
```bash
black src/
ruff check src/
mypy src/tsdynamics/
pytest -q
```

### Writing `_rhs` for `DynSys` / `DynSysDelay`

- Use only **symbolic expressions** — no NumPy, no `math`, no Python `if`
- Access state with `y(i)` (ODE) or `y(i, t - tau)` (DDE)
- Keep expressions O(N) in the dimension count
- For periodic-boundary systems, use modular index arithmetic: `y((i+1)%N)`
- Return a **list or tuple** of exactly `n_dim` expressions

### Writing `_rhs` and `_jac` for `DynMap`

- Decorate with `@staticjit` (numba JIT, gracefully degrades)
- Params arrive as **positional args** in `params` dict-insertion order
- Return array-like of length `n_dim`

### Parameter management

- Class-level `params` dict defines defaults
- Constructor accepts `params` override as dict
- All params are also accessible as attributes; writes sync back to `params`
- Pass `initial_conds` at class level, constructor, or integrate time (later takes precedence)

### Initial conditions

- If `None` at integration time, random `U[0,1)^n_dim` is used and stored in `self.initial_conds`
- Passing `initial_conds` to `integrate` always overwrites `self.initial_conds`
- For DDEs: provide a non-trivial `history` callable to avoid equilibrium pasts that kill dynamics

## Testing

```bash
pytest -q
pytest --cov=tsdynamics --cov-report=term-missing
```

Key testing rules:
- For Lyapunov exponents: compare to **ranges**, not exact scalars (convergence varies with tolerances and `dt`)
- Suppress expected warnings **locally** with `warnings.catch_warnings()`, never globally in the test
- DDEs: always provide a nontrivial `history`; equilibrium histories yield trivial exponents

## Adding a new system

### New ODE system

1. Pick the right module under `systems/continuous/` by topic
2. Subclass `DynSys`, set `params` and `n_dim`
3. Implement `@staticmethod _rhs(y, t, **params)` returning symbolic expressions
4. Optionally add `@staticmethod _jac` for stiff integrators
5. Export from the module's `__init__.py`

```python
# Example skeleton
from tsdynamics.base.ode_base import DynSys

class MyAttractor(DynSys):
    params = {"a": 1.0, "b": 2.0, "c": 3.0}
    n_dim = 3

    @staticmethod
    def _rhs(y, t, a, b, c):
        x, yv, z = y(0), y(1), y(2)
        return (
            a * (yv - x),
            x * (b - z) - yv,
            x * yv - c * z,
        )
```

### New DDE system

Same as ODE but:
- Subclass `DynSysDelay`
- Use `y(i, t - tau_value)` for delayed terms
- Provide example `history` callable in docstring/tests

### New discrete map

1. Pick the right module under `systems/discrete/`
2. Subclass `DynMap`, set `params` and `n_dim`
3. Implement both `_rhs` and `_jac` as `@staticjit` static methods
4. Params arrive positionally in dict-insertion order — match the signature accordingly

```python
from tsdynamics.base.map_base import DynMap
from tsdynamics.utils.general import staticjit

class MyMap(DynMap):
    params = {"a": 1.4, "b": 0.3}
    n_dim = 2

    @staticjit
    def _rhs(X, a, b):
        x, y = X
        return (1 - a*x**2 + y, b*x)

    @staticjit
    def _jac(X, a, b):
        x, y = X
        return [[-2*a*x, 1], [b, 0]]
```

### New utility

- Add to `utils/` as a standalone module
- Export from `utils/__init__.py`
- Return a `dataclass` for rich, inspectable results

## Common pitfalls

| Situation | What happens / what to do |
|---|---|
| `_rhs` uses NumPy inside | JiTCODE can't compile it — use symbolic ops only |
| DDE with trivial constant history at fixed point | Lyapunov exponents come out ~0 — provide nontrivial `history` |
| Both `steps` and `final_time` passed | `steps` wins, a print warning is emitted |
| Map `_rhs` / `_jac` params wrong order | Parameters arrive positionally; must match `params` dict-insertion order |
| Tight tolerances on DDE | Start with `rtol=atol=1e-3`; DDEs are stiff — very tight tolerances can stall |
| `n_lyap` not set for DDE Lyapunov | Defaults to `1`; DDEs have infinitely many exponents — choose consciously |
| Forgetting `@staticjit` on map methods | Numba won't accelerate it; still works, just slower |
| KuramotoSivashinsky N < 5 | Constructor raises `ValueError` |

## Quick reference

```python
import numpy as np
from tsdynamics.systems.continuous.chaotic_attractors import Lorenz
from tsdynamics.systems.continuous.delayed_systems import MackeyGlass
from tsdynamics.systems.discrete.chaotic_maps import Henon
from tsdynamics.viz.plotters import phase_portrait, trajectory3d
from tsdynamics.viz.base import PlotConfig

# ODE
lor = Lorenz()
t, X = lor.integrate(dt=0.01, final_time=100.0, method="dop853", rtol=1e-8, atol=1e-10)
exps = lor.lyapunov_spectrum(dt=0.1, burn_in=50.0, final_time=300.0)
# → ~[0.91, ~0, -14.57]

# DDE
mg = MackeyGlass()
hist = lambda s: [1.0 + 0.1 * np.sin(0.2 * s)]
t, y = mg.integrate(dt=0.05, final_time=500.0, history=hist, rtol=1e-6, atol=1e-9)
exps = mg.lyapunov_spectrum(n_lyap=1, dt=0.2, burn_in=100.0, final_time=600.0, history=hist)

# Map
h = Henon()
t_idx, traj = h.iterate(steps=5000)
exps = h.lyapunov_spectrum(steps=5000)

# Plot
cfg = PlotConfig(figsize=(8, 6), dpi=150)
fig, ax = phase_portrait(X, dims=(0, 2), cfg=cfg)
fig, ax = trajectory3d(t, X, dims=(0, 1, 2), cfg=cfg)
```
