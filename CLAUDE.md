## Project overview

**TSDynamics** is a Python library for studying dynamical systems. It provides:

- Compiled ODE integration via JiTCODE (`ContinuousSystem`)
- Compiled DDE integration via JiTCDDE (`DelaySystem`)
- Discrete-map iteration via Numba (`DiscreteMap`)
- Lyapunov spectra for all three families
- A handful of timestep-estimation utilities

The user defines the math (one symbolic `_equations` method, optionally
`_jacobian`); the library handles compilation, caching, integration, and
output grids.

**Author:** Daniel Estevez
**Python:** ≥ 3.12
**Package manager:** uv (recommended)
**License:** MIT

## Public API surface

`tsdynamics.__all__` exports:

- Every built-in system class (`Lorenz`, `Rossler`, ..., `MackeyGlass`, ..., `Henon`, ...)
- The three subclassable base classes: `ContinuousSystem`, `DelaySystem`, `DiscreteMap`
- The result type: `Trajectory`
- Submodules: `analysis`, `base`, `systems`, `utils`
- `__version__`

Not exported at the top level (deliberately) but reachable via `tsdynamics.base` / `tsdynamics.utils`:

- `SystemBase` — abstract common base; users do not subclass this directly
- `ParamSet` — fixed-key parameter container (`system.params` is an instance of this)
- `staticjit` — Numba decorator for `_step` / `_jacobian` on maps

## Base classes

### `SystemBase` (`base/base.py`)

Shared parameter / IC / metadata machinery.

```python
class SystemBase:
    params: ClassVar[dict] = {}              # class-level defaults
    dim: ClassVar[int | None] = None         # state-space dimension
    default_ic: ClassVar[Any | None] = None  # optional class-level IC
```

Instance attributes after `__init__`:

- `self.params` — a `ParamSet` (MutableMapping with fixed keys, dict-style + attribute-style access). Attribute writes on the system are forwarded to `self.params`.
- `self.dim` — int.
- `self.ic` — `np.ndarray | None`. Set by `resolve_ic(...)`.
- `self.meta` — empty dict for caching computed properties (e.g. `meta["lyapunov_spectrum"]`).

IC resolution priority in `resolve_ic(ic=None)`:
1. `ic` argument
2. `self.ic`
3. `type(self).default_ic`
4. random `U[0, 1)^dim`

Cloning helpers: `system.copy()`, `system.with_params(**overrides)`.

### `ContinuousSystem` (`base/ode_base.py`)

ODE integration via JiTCODE.

**Subclass contract:**

```python
class MyODE(ContinuousSystem):
    params = {"a": 1.0, "b": 2.0}
    dim = 2

    @staticmethod
    def _equations(y, t, *, a, b):     # kwargs, names must match params keys
        return (a * y(0) - y(1), b * y(0) + y(1))

    @staticmethod                       # optional — helpful for stiff integrators
    def _jacobian(y, t, *, a, b):
        return [[a, -1], [b, 1]]
```

- `y(i)` — JiTCODE symbol for state component `i`.
- `t` — JiTCODE time symbol.
- Use **only** symbolic operations (`symengine.sin`, `symengine.cos`, `+`, `*`, `**`, `abs`, ...). No NumPy, no `math`, no Python `if`/`for` over `y`.
- Return a sequence of exactly `dim` expressions.

**Variable-dimension systems** (Lorenz-96, KS, MultiChua) declare any parameter that drives the *symbolic structure* (e.g. a loop length) as **structural**:

```python
class Lorenz96(ContinuousSystem):
    params = {"f": 8.0, "N": 20}
    _structural_params = frozenset({"N"})   # baked in at compile time
```

Structural params are baked into the compiled `.so`; non-structural params become JiTCODE `control_pars` and can be changed at runtime without recompiling. The cache key is `(class, dim, hash(structural_params))`.

**Integration:**

```python
sys.integrate(
    final_time=100.0,
    dt=0.02,            # output grid; the internal stepper is adaptive
    *,
    t0=0.0,
    ic=None,
    method="RK45",      # also: "dopri5", "DOP853", "LSODA", "VODE"
    rtol=1e-6, atol=1e-9,
    **integrator_kwargs,   # forwarded to jitcode.set_integrator
) -> Trajectory
```

**Lyapunov:**

```python
sys.lyapunov_spectrum(
    final_time=200.0, dt=0.1,
    *,
    ic=None,
    n_exp=None,             # defaults to dim
    burn_in=50.0,
    method=None, rtol=1e-6, atol=1e-9,
    **integrator_kwargs,
) -> np.ndarray             # shape (n_exp,), descending
```

Uses `jitcode_lyap`. The per-interval local exponents are time-weight averaged.

### `DelaySystem` (`base/dde_base.py`)

DDE integration via JiTCDDE.

**Subclass contract:**

```python
class MyDDE(DelaySystem):
    params = {"k": 2.0, "tau": 1.5}
    dim = 1
    _delay_params = ("tau",)   # default — override for non-"tau" names or multi-delay

    @staticmethod
    def _equations(y, t, k, tau):
        return [k * y(0, t - tau) - y(0)]
```

- Use `y(i, t - tau_expr)` for delayed access.
- `_delay_params` lists which parameters are delays (positive floats); these are passed to JiTCDDE's `delays=` and used to size the spline buffer. Override `_delays()` for delays computed from non-parameter expressions.
- Compilation caches on `(class, hash(all params))` — every parameter change triggers a recompile because delay structure can change.

**Integration / Lyapunov** have the same signature as `ContinuousSystem` but with `history` (a callable `h(s) -> sequence`) instead of `t0`:

```python
mg = MackeyGlass()
hist = lambda s: [1.0 + 0.1 * np.sin(0.2 * s)]
traj = mg.integrate(final_time=500.0, dt=0.5, history=hist)
# DDE Lyapunov needs a state on the attractor — run integrate first:
exps = mg.lyapunov_spectrum(n_exp=1, dt=0.5, ic=traj.y[-1])
```

DDE-specific notes:
- Default tolerances are `1e-3` (DDEs are stiff; tighter values often stall the solver).
- `lyapunov_spectrum` cannot take a `history` callable — JiTCDDE's `jitcdde_lyap` is incompatible with `past_from_function`. Pass `ic=traj.y[-1]` from a prior `integrate` call to start on the attractor.

### `DiscreteMap` (`base/map_base.py`)

Discrete-map iteration with QR-based Lyapunov spectrum.

**Subclass contract:**

```python
from tsdynamics.utils import staticjit

class MyMap(DiscreteMap):
    params = {"a": 1.4, "b": 0.3}
    dim = 2

    @staticjit
    def _step(X, a, b):                 # positional args, params in dict-insertion order
        x, y = X
        return (1 - a*x**2 + y, b*x)

    @staticjit
    def _jacobian(X, a, b):
        x, y = X
        return ((-2*a*x, 1.0), (b, 0.0))
```

`@staticjit` applies Numba `njit` + `staticmethod`, gracefully degrading when Numba is not installed.

⚠ **Parameter order matters.** `_step` and `_jacobian` receive params *positionally* in the order they appear in `params`. Keep the function signature and the `params` dict insertion order in sync — `Circle._step(X, k, omega)` with `params={"omega": 0.333, "k": 5.7}` previously silently swapped them.

`iterate(steps=1000, ic=None, max_retries=10) -> Trajectory`
`lyapunov_spectrum(steps=5000, ic=None, n_exp=None, reortho_interval=1) -> ndarray`

The Lyapunov loop runs under `np.errstate(over="ignore", invalid="ignore")` and falls back to a new random IC if the iterate diverges, so transient overflow on huge iterates doesn't crash the whole call.

## Adding a new system

### A new ODE system

1. Drop the class into the right module under `systems/continuous/`.
2. Subclass `ContinuousSystem`. Declare `params` and `dim`.
3. Implement `@staticmethod _equations(y, t, **params)` with **symbolic** ops.
4. Optionally add `_jacobian` (helpful for stiff integrators).
5. If any parameter drives loop length / list comprehension structure, add it to `_structural_params`.
6. Add the class name to the module's `__all__` and the parent package's `__init__.py`.
7. Add a row to `tests/test_ode_systems.py` (`ALL_ODE_SYSTEMS`) and optionally to `_INTEGRATION_SAMPLE`.

### A new DDE system

Same as ODE but subclass `DelaySystem`, use `y(i, t - tau)` for delayed access, and override `_delay_params` if any delay parameter isn't called `"tau"`.

### A new discrete map

1. Subclass `DiscreteMap` under the right `systems/discrete/` module.
2. Decorate `_step` and `_jacobian` with `@staticjit`.
3. Make sure the function signature matches the `params` dict insertion order.
4. If random `U[0, 1)^dim` ICs don't land in the attractor basin, set `default_ic` as a class attribute.

## Compilation cache

Compiled JiTCODE/JiTCDDE shared objects live in `~/.cache/tsdynamics/` (override with `TSDYNAMICS_CACHE`). The first integration of a system compiles to a `.so`; subsequent calls (and Python sessions) reuse the cached binary.

- ODE cache key: `tsdyn_<ClassName>_<dim>[_<hash-of-structural-params>]`
- DDE cache key: `tsdyn_dde_<ClassName>_<hash-of-all-params>` (DDE structure depends on all params)

Wipe the cache (`rm -rf ~/.cache/tsdynamics/`) after changing an `_equations` body or after major JiTCODE upgrades.

## Common pitfalls

| Situation | What happens / what to do |
|---|---|
| `_equations` uses NumPy or `math` | JiTCODE can't symbolic-compile it. Use `symengine.sin`/`cos`/... |
| Variable-dim system without `_structural_params` | Loop bound is a SymEngine `Symbol`; compile-time `range(N)` fails. Add `_structural_params = frozenset({"N"})`. |
| DDE with a trivial constant past at a fixed point | All Lyapunov exponents come out ≈ 0. Provide a non-equilibrium `history`. |
| Tight tolerances on DDE | `rtol=atol=1e-3` is the safe starting point. |
| Map params arrive in the wrong order in `_step` / `_jacobian` | Match the `params` dict insertion order. |
| Forgetting `@staticjit` on map methods | Still works, just slower; Numba's JIT not applied. |
| Stale compiled cache after editing `_equations` | Wipe `~/.cache/tsdynamics/` and re-run. |
| KS / Lorenz-96 with very small `N` | KS raises `ValueError` for `N < 7`. |

## Quick reference

```python
import numpy as np
import tsdynamics as ts

# ODE
lor = ts.Lorenz()
traj = lor.integrate(final_time=100.0, dt=0.01, method="dop853", rtol=1e-8, atol=1e-10)
exps = lor.lyapunov_spectrum(dt=0.1, burn_in=50.0, final_time=300.0)
# → [0.91, ~0, -14.57]

# Variable-dim ODE (Lorenz-96)
l = ts.Lorenz96(N=10, f=8.0)
traj = l.integrate(final_time=50.0, dt=0.05)

# DDE
mg = ts.MackeyGlass()
hist = lambda s: [1.0 + 0.1 * np.sin(0.2 * s)]
traj = mg.integrate(final_time=500.0, dt=0.5, history=hist)
exps = mg.lyapunov_spectrum(n_exp=1, dt=0.5, ic=traj.y[-1], burn_in=100.0, final_time=1000.0)

# Map
h = ts.Henon()
traj = h.iterate(steps=5000)
exps = h.lyapunov_spectrum(steps=5000)        # → [0.42, -1.62]

# Trajectory enrichment (M1) — every method returns a fresh Trajectory
traj = ts.Lorenz().integrate(final_time=200, dt=0.005).after(50.0)
sparse  = traj.decimate(every=10)
uniform = traj.resample(dt_new=0.01, kind="cubic")
xz      = traj.project(dims=(0, 2))
middle  = traj.window(t0=100.0, t1=150.0)
dtraj   = traj.derivative(order=1)
rtraj   = traj.norm()                              # Trajectory, y.shape == (T, 1)
peaks   = traj.local_maxima(component=2, prominence=1.0)            # Trajectory
peaksR  = traj.local_maxima(component=2, refined=True)              # sub-sample Hermite refinement
isi     = traj.return_times(component=2, prominence=1.0)            # Trajectory of ISIs
spec    = traj.to_dataspec(kind="phase_portrait_3d", dims=(0, 1, 2))

# Event & section detection (M2) — Poincaré, return maps, threshold crossings.
# Three call styles per op: shortcut kwargs, condition object, or bare callable.
from tsdynamics.analysis import Plane, LinearPlane                  # only two condition classes

# 1. Shortcut kwargs (the easy one)
events = traj.detect_events(axis=0, value=10.0, direction="up")
sec    = traj.poincare_section(axis=2, value=27.0)                  # direction="up" by default
rmap   = traj.return_map(axis=2, value=27.0, observable=0)

# 2. Condition objects (for power users / LinearPlane case)
events = traj.detect_events(Plane(axis=0, value=10.0, direction="up"))
sec    = traj.poincare_section(LinearPlane(normal=[1, 1, 0], offset=0))

# 3. Bare callable
events = traj.detect_events(lambda t, y: float(np.linalg.norm(y) - 1.0), direction="up")

# Every result is a Trajectory.  To plot the canonical x_{k+1} vs x_k return map:
spec   = rmap.to_dataspec(kind="return_map", step=1)                # {kind, x, y, t, step, observable}
# plt.scatter(spec["x"], spec["y"])
```

### Two call forms, one implementation — the `@trajectory_op` registry

There is **exactly one definition** of each analysis primitive.  It's a
free function

```python
def decimate(t, y, every): ...                         # in _trajectory_ops.py
```

decorated with `@trajectory_op(returns=...)` from
`tsdynamics.analysis._registry`.  The decorator does two things at
import time:

1. Wraps the function so callers can pass a `Trajectory`, a `(t, y)`
   tuple, or bare `t, y` arrays as the leading argument(s).
2. Records the function in an internal registry.

When `Trajectory` is defined, the very last line of
`src/tsdynamics/base/base.py` calls
`install_methods(Trajectory)` which drains the registry and attaches one
method per entry on the class.  The method strips `self.t, self.y` from
its arguments and forwards to the function.

The net effect — **every primitive is callable both ways without any
hand-written wrapper anywhere**:

```python
traj.decimate(every=5)                        # method form
decimate(traj, every=5)                       # free function with Trajectory
decimate((t, y), every=5)                     # free function with tuple
decimate(t, y, every=5)                       # free function with raw arrays

traj.detect_events(Plane(axis=2, value=27))   # method form
detect_events(traj, Plane(axis=2, value=27))  # free function form
```

`returns=` controls the wrapping:

| `returns=`        | What the fn returns                  | Wrapped to                              |
|-------------------|--------------------------------------|-----------------------------------------|
| `"trajectory"`     | `(t_new, y_new)` tuple               | `Trajectory(t_new, y_new, system)`      |
| `"ndarray_keep_t"` | ndarray with same time axis as input | `Trajectory(self.t.copy(), arr, system)`|
| `"passthrough"`    | anything else (only `to_dataspec` uses this — returns a plain dict that V1's `DataSpec` will replace) | returned as-is |

`Trajectory.system` is inherited from the input when the input is a
`Trajectory`, and is `None` when the caller supplied raw arrays / a
`(t, y)` tuple.  `Trajectory` is tuple-unpackable so call sites that
prefer `t, y = decimate(...)` still work transparently.

### Adding a new analysis primitive

Drop the function next to its peers in
`src/tsdynamics/analysis/_trajectory_ops.py` (or `_events.py` if it's
section / event / return-map shaped), give it a `(t, y, *args)`
signature, decorate with `@trajectory_op(returns=...)`, re-export it
from `analysis/__init__.py`.  That's it.  No class to touch, no
docstring to duplicate, no wrapper method to keep in sync.

**Return-type contract.** The analysis module ships a single output
type — :class:`Trajectory`.  Every primitive returns one, including the
reductions (`norm`, `local_maxima`, `local_minima`, `return_times`) for
which the output is a 1-column trajectory with `y.shape == (K, 1)`.
The single non-Trajectory exit is `to_dataspec`, which builds the
placeholder dict V1 will turn into a real `DataSpec`.

```python
@trajectory_op(returns="trajectory")
def smooth(t, y, window: int = 5):
    """Single docstring lives here — surfaces in both forms."""
    kernel = np.ones(window) / window
    y_new = np.apply_along_axis(
        lambda col: np.convolve(col, kernel, mode="same"), axis=0, arr=y,
    )
    return t.copy(), y_new
```

After re-exporting, both `traj.smooth(window=5)` and
`smooth(traj, window=5)` work, with the right return type, automatically.
